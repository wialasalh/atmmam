# بنية لوحة أتمم

الواجهة الحالية تعمل ببيانات تجريبية محفوظة في المتصفح حتى يمكن اعتماد تجربة الاستخدام. بنية الإنتاج المستهدفة:

- Next.js على Vercel بوضع ديناميكي للوحة، مع بقاء صفحات الموقع العامة قابلة للتخزين المؤقت.
- Supabase PostgreSQL للطلبات والعملاء والخدمات والمتابعات وسجل النشاط.
- Supabase Auth للمستخدمين، مع أدوار: مدير النظام، مدير العمليات، موظف عمليات، ومشاهد.
- Supabase Storage لمستندات الطلبات، مع مسارات خاصة وسياسات وصول مرتبطة بالموظفين.
- كل تغيير مهم يُسجل في `order_activity` و`audit_logs`.
- الحذف التشغيلي يكون Soft Delete للحفاظ على التاريخ.
- في بيئة الإنتاج تفشل حماية لوحة الإدارة بشكل مغلق: إذا غابت مفاتيح Supabase فلن تُفتح مسارات التشغيل.

لا يُستخدم مفتاح `SUPABASE_SERVICE_ROLE_KEY` في المتصفح مطلقًا.
