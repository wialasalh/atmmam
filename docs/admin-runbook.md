# دليل تشغيل لوحة أتمم

## قبل النشر

- شغّل `pnpm check` وتأكد من نجاح الأنواع والاختبارات والبناء.
- طبّق جميع migrations على بيئة تجريبية أولًا ثم الإنتاج.
- اختبر أدوار admin وmanager وoperator وviewer بحسابات منفصلة.
- تأكد أن مستودع `order-documents` خاص وأن روابط الملفات مؤقتة.
- لا تضف Service Role Key إلى متغيرات `NEXT_PUBLIC_*`.

## التشغيل اليومي

- الطلب لا ينتقل إلى مكتمل قبل مراجعة المستندات والإجراء التالي.
- الحالات المتعثرة أو الملغاة تتطلب سببًا مسجلًا.
- لا تحذف العميل أو الطلب نهائيًا؛ استخدم الحذف الناعم.
- راجع المتابعات المتأخرة وسجل التدقيق يوميًا.

## النسخ والاستعادة

- فعّل نسخ قاعدة البيانات الدورية في Supabase.
- اختبر الاستعادة دوريًا على مشروع منفصل.
- احتفظ بالمستندات وسياسات Storage ضمن خطة الاستعادة.

## حادث أمني

1. عطّل حساب الموظف المتأثر من Auth و`profiles.active`.
2. دوّر المفاتيح المتأثرة من Supabase وVercel.
3. راجع `audit_logs` و`order_activity` وحدد نطاق الحادث.
4. لا تعدّل سجل التدقيق؛ صدّر نسخة للتحقيق.
